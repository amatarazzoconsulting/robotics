# Preface

This book is the culmination of a vision: that robotics should not be a collection of expensive, brittle, single-purpose machines, but rather a unified ecosystem where intelligence lives in the cloud (or your smart TV) and movement is generated by a common, efficient, real-time language called **lowL**.

We call this vision **Algorithmic Movement** – the principle that every motion, from the precise arc of an industrial arm to the joyful sway of a child's toy dancer, is the direct execution of an algorithm. The algorithm may be simple (a reflex to avoid a cliff) or complex (a GPT-generated choreography for ten robots), but the underlying execution engine is the same.

# Volume I: Foundations

## Chapter 1: The Philosophy of Algorithmic Movement

### 1.1 What is Algorithmic Movement?

Algorithmic Movement is a paradigm shift in robotics. Traditional robotics embeds intelligence inside each robot – expensive processors, complex sensors, and proprietary software. This makes robots costly, difficult to update, and often obsolete within months.

Algorithmic Movement inverts this model:

> **Intelligence is centralized. Movement is distributed.**

A central AI server (often the family's smart TV, a home hub, or a cloud instance) handles all complex computation: perception, planning, learning, and choreography. Each robot is a **thin client** – a simple, low-cost functional unit that receives movement commands and executes them with real-time precision.

This model, enabled by the **lowL compiler** and **AlgorithmicOS**, delivers:
- **90% cost reduction** per robot (intelligence is shared across many units)
- **Instant updates** (the server improves; all robots improve)
- **Perfect synchronization** (one server, one clock, many robots)
- **Unlimited complexity** (server can run GPT-5; robot just needs 256 KB of flash)

### 1.2 The Three Pillars

**Pillar 1: Centralized Intelligence**
The AI server (OpenAI, LLaMA, or custom model) runs on an x86_64 machine with abundant RAM, GPU acceleration, and persistent storage. It perceives the world through cameras (on the TV or separate sensors), understands natural language commands, generates movement plans, and streams commands to robots.

**Pillar 2: Thin Client Robots**
Each robot contains only:
- A low-cost Intel x86_64 microcontroller (or compatible)
- Basic actuators (servos, motors)
- Minimal sensors (optional)
- Wireless receiver
- Battery or USB power

No camera. No complex AI. No expensive compute. Just a reliable executor of movement commands.

**Pillar 3: The lowL Bridge**
The lowL compiler translates high-level movement descriptions (JSON from the AI server) into efficient, real-time machine code for each robot's specific actuator configuration. AlgorithmicOS, written entirely in lowL, runs on every robot, providing:
- Real-time wireless command reception
- Deterministic servo/motor control
- Safety monitoring and fallback
- Low-power idle modes

### 1.3 The Economic Revolution

| Traditional Robot | Algorithmic Movement Robot |
|-------------------|----------------------------|
| $500–$2000 | $15–$100 |
| 2-year obsolescence | Indefinite (server updates) |
| One robot per family | Swarm of 10+ robots |
| Proprietary software | Open source (lowL + AlgorithmicOS) |
| Difficult to repair | Simple, modular design |

### 1.4 Ethical Design Principles

Algorithmic Movement is not just about cost and performance – it is about building robots that serve humans, not the other way around.

**Principle 1: Privacy by Default**
No robot sends video, audio, or location data to the cloud without explicit user consent. The AI server (if local) keeps all data in the home.

**Principle 2: Safety as Priority Zero**
All robots have hardware and software emergency stops. Movement commands are validated by `when` guards before execution.

**Principle 3: Transparency**
Users can inspect the lowL firmware running on any robot. No hidden backdoors, no proprietary black boxes.

**Principle 4: Accessibility**
Robots should be affordable for all families, not just wealthy early adopters. The thin client model democratizes robotics.

**Principle 5: Joy**
Robots should make people smile. The dancing toy robots are a reminder that technology can be delightful, not just efficient.

### 2.3 Common Hardware Abstraction Layer

AlgorithmicOS provides a uniform hardware abstraction layer (HAL) for all robots, written in lowL:

```lowl
// hal.lowl - Hardware Abstraction Layer

// Servo abstraction
struct Servo:
    pin: u8
    min_pulse_us: u16 = 500
    max_pulse_us: u16 = 2500
    current_angle: u8  // 0-255

impl Servo:
    fn write_angle(angle: u8):
        let pulse_us = this.min_pulse_us + (angle as u16) * 
                       (this.max_pulse_us - this.min_pulse_us) / 255
        set_pwm(this.pin, pulse_us)
        this.current_angle = angle

// Motor abstraction (differential drive)
struct DifferentialDrive:
    left_motor: Motor
    right_motor: Motor
    wheel_diameter_mm: u16
    wheel_base_mm: u16

impl DifferentialDrive:
    fn drive(speed_left: i8, speed_right: i8):  // -127 to 127
        this.left_motor.set_speed(speed_left)
        this.right_motor.set_speed(speed_right)
    
    fn turn(degrees: i16, speed: u8):
        // Pure rotation turn implementation
        let duration_ms = (degrees as i32).abs() * this.wheel_base_mm / 
                          (speed as i32) * 10
        this.drive(speed as i8, -(speed as i8))  // opposite directions
        delay_ms(duration_ms)
        this.stop()
```

## Chapter 3: Introducing lowL – The Systems Language

### 3.1 Why a New Language?

C gives you control but lacks safety. C++ gives you abstractions but hides costs. Rust gives you safety but imposes a complex ownership model. **lowL** was designed specifically for Algorithmic Movement:

- **Compiles to x86_64 assembly** (no runtime, no garbage collector)
- **Zero-cost abstractions** (classes, templates, pattern matching compile to minimal code)
- **Hardware builtins** (`port_write8`, `read_cr2`, `rdtsc`, etc.)
- **SIMD first-class** (`vec4_f32`, `vec8_f32`, `vec16_f32`)
- **Python-style indentation** (clean, readable systems code)
- **Deterministic performance** (no hidden allocations, no exceptions)

### 3.2 lowL at a Glance

```lowl
// A complete lowL program for a dancing robot
fn main() -> u32:
    init_servos()
    init_wifi()
    
    let mut pose = idle_pose()
    
    while true:
        let packet = wifi_receive()
        if packet.len() > 0:
            let new_pose = parse_command(packet)
            if new_pose.is_some():
                pose = new_pose.unwrap()
        
        execute_pose(pose)
        delay_ms(20)  // 50 Hz update
    
    return 0

fn execute_pose(pose: Pose):
    for i in 0..4:
        servos[i].write_angle(pose.angles[i])
    set_led(pose.led_color)
```

```lowl
fn fuse_imu(gyro: vec4_f32, accel: vec4_f32) -> vec4_f32:
    // Complementary filter with SIMD
    let alpha = 0.98
    let angle = gyro * alpha + accel * (1.0 - alpha)
```

```lowl
            return Option.some(Command.DanceSequence(packet[1]))

            return Option.none()
```

### 3.4 The lowL Compiler Pipeline

```
Source (.lowl) → Lexer → Parser → AST Optimizer → Code Generator → Assembly (.asm) → NASM → Binary
```

Optimization levels:
- **-O0:** No optimization (debugging)
- **-O1:** Constant folding, dead code elimination
- **-O2:** Loop invariant motion, basic SIMD (default)
- **-O3:** Loop unrolling, aggressive SIMD, function inlining

## Chapter 4: AlgorithmicOS – The Operating System for Robots

### 4.1 Architecture Overview

AlgorithmicOS is a minimal, real-time operating system written entirely in lowL. It provides:

| Component | Description |
|-----------|-------------|
| **Scheduler** | Cooperative round-robin (deterministic, low latency) |
| **Wireless Stack** | Lightweight UDP/IP over Wi-Fi or BLE |
| **Command Parser** | Pattern-matching dispatcher for incoming commands |
| **Actuator Manager** | Real-time servo/motor control with interpolation |
| **Safety Monitor** | Watchdog, emergency stop, timeout detection |
| **Power Manager** | Sleep modes, battery monitoring, charge control |
| **Update Manager** | Over-the-air firmware updates (tiny, differential) |

### 4.2 Memory Layout

```
┌─────────────────────────────────────────────┐ 0x00000000
│  Interrupt Vector Table                      │
├─────────────────────────────────────────────┤
│  lowL Firmware Text (.text)                  │
├─────────────────────────────────────────────┤
│  Read-Only Data (.rodata)                    │
│  - Dance data sections                       │
│  - Command tables                            │
├─────────────────────────────────────────────┤
│  Initialized Data (.data)                    │
├─────────────────────────────────────────────┤
│  Zero-Initialized Data (.bss)                │
├─────────────────────────────────────────────┤
│  Heap (optional, often disabled)             │
├─────────────────────────────────────────────┤
│  Stack (grows downward)                      │
└─────────────────────────────────────────────┘ 0x0003FFFF (256 KB)
```

### 4.3 Task Management

AlgorithmicOS uses a simple cooperative task model (no preemption, no mutexes needed):

```lowl
// task.lowl - Cooperative task scheduler

struct Task:
    fn() -> void   // function pointer
    period_ms: u16
    last_run: u32  // timestamp in ms

static tasks: array<Task, MAX_TASKS>
static task_count: u64 = 0

fn add_task(task_fn: fn() -> void, period_ms: u16) -> bool:
    if task_count >= MAX_TASKS:
        return false
    tasks[task_count] = Task{task_fn, period_ms, 0}
    task_count = task_count + 1
    return true

fn scheduler_main():
    while true:
        let now = get_tick_ms()
        for i in 0..task_count:
            if (now - tasks[i].last_run) >= tasks[i].period_ms:
                tasks[i].fn()
                tasks[i].last_run = now
        // Yield CPU (optional)
        pause()
```

### 4.4 Wireless Command Protocol

AlgorithmicOS implements a lightweight **UDP-based protocol** for receiving commands from the AI server:

**Packet Format:**
```
┌──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┐
│0xAA  │ Len  │ Seq  │ Cmd  │ Payload...          │ CRC  │
│1 byte│1 byte│1 byte│1 byte│0-32 bytes           │1 byte│
└──────┴──────┴──────┴──────┴─────────────────────┴──────┘
```

**Command Types:**
| Cmd | Name | Payload | Description |
|-----|------|---------|-------------|
| 0x01 | Ping | none | Health check |
| 0x02 | Pose | 4x u8 angles | Immediate servo positions |
| 0x03 | Motion | pose1, pose2, duration | Interpolate between poses |
| 0x04 | Sequence | index (0-255) | Execute pre-stored dance |
| 0x05 | Stop | none | Emergency stop |
| 0x06 | Sleep | none | Low power mode |
| 0x07 | Wake | none | Exit sleep |

### 4.5 Safety Subsystem

The safety monitor runs at the highest interrupt priority (`#[interrupt]` with `priority = 0`):

```lowl
#[interrupt(priority = 0)]  // Highest priority
fn safety_monitor():
    // Check watchdog (server must send keepalive every 5 seconds)
    if (get_tick_ms() - last_keepalive) > 5000:
        emergency_stop()
    
    // Check battery voltage (if ADC available)
    let battery_mv = read_battery_voltage()
    if battery_mv < 3200:  // 3.2V critical
        emergency_stop()
    
    // Check motor current (overload protection)
    if read_motor_current() > MAX_CURRENT_MA:
        disable_motors()
    
    // Check for physical stall (encoder not moving)
    if commanded_movement and not encoder_changed():
        disable_motors()
    
    // Re-enable interrupts for lower priority tasks
    enable_interrupts()
```

### 4.6 Power Management

```lowl
// powermgr.lowl

fn sleep_mode():
    // Save current pose to restore later
    let saved_pose = get_current_pose()
    
    // Disable Wi-Fi (power save)
    wifi_power_down()
    
    // Reduce clock speed (if supported)
    set_cpu_freq(32)  // MHz
    
    // Configure wake sources (wireless, button, timer)
    enable_wake_on_wifi()
    enable_wake_on_button()
    set_wake_timer(3600000)  // 1 hour max sleep
    
    // Enter halt with interrupts enabled
    enable_interrupts()
    while true:
        halt()  // CPU stops until interrupt
        if should_wake():
            break
    
    // Restore
    wifi_power_up()
    set_cpu_freq(CPU_FREQ_MAX)
    execute_pose(saved_pose)
```

### 5.6 lowL Firmware for Spark (Complete Source)

```lowl
// spark_firmware.lowl
// Complete firmware for the Spark Dancing Robot
// Compile: lowlc spark_firmware.lowl -f elf -O2

const VERSION: string = "Spark 1.0"
const WIFI_SSID: string = "Spark_Network"
const WIFI_PASS: string = "dance123"
const SERVER_PORT: u16 = 8888
const SERVER_IP: u32 = 192 << 24 | 168 << 16 | 1 << 8 | 100  // 192.168.1.100
const MAX_PACKET_SIZE: u64 = 64
const HEARTBEAT_TIMEOUT_MS: u64 = 5000

// Servo configuration (PWM values at 50 Hz)
struct ServoCal:
    pin: u8
    min_us: u16 = 500
    max_us: u16 = 2500
    neutral_us: u16 = 1500

const SERVOS: array<ServoCal, 4> = [
    ServoCal{0, 500, 2500, 1500},  // Head
    ServoCal{1, 500, 2500, 1500},  // Left arm
    ServoCal{2, 500, 2500, 1500},  // Right arm
    ServoCal{3, 500, 2500, 1500}   // Body spin
]

// Robot state
struct RobotState:
    angles: array<u8, 4>
    led_rgb: array<u8, 3>
    last_heartbeat: u32
    is_dancing: bool

// Command types
enum CommandType:
    Ping = 0x01
    Pose = 0x02
    Motion = 0x03
    Sequence = 0x04
    Stop = 0x05
    Sleep = 0x06
    Wake = 0x07

// Dance sequences (stored in .rodata via data section)
data_section format SparkDances columnar:
    record DanceSequence:
        name: string
        angles1: array<u8, 4>
        angles2: array<u8, 4>
        duration_ms: u16

    key(name) -> string rb_map as "dance_lookup"

records:
    "idle",    [90, 0, 0, 0],    [90, 0, 0, 0],    0
    "wave",    [90, 0, 0, 0],    [90, 90, 90, 0], 1000
    "spin",    [90, 0, 0, 0],    [90, 0, 0, 180], 1500
    "bow",     [90, 0, 0, 0],    [45, 0, 0, 0],   800
    "cheer",   [90, 0, 0, 0],    [135, 90, 90, 0], 600
end

// Global state
static state: RobotState
static wifi_connected: bool = false

// ============================================================================
// Initialization
// ============================================================================

fn main() -> u32:
    init_servos()
    init_leds()
    init_wifi()
    init_timer()
    
    // Set initial state
    state.angles = [90, 0, 0, 0]
    state.led_rgb = [0, 255, 0]  // Green
    state.last_heartbeat = get_tick_ms()
    state.is_dancing = false
    
    execute_pose(state.angles, state.led_rgb)
    
    // Register safety monitor
    add_interrupt_handler(IRQ_TIMER, safety_monitor)
    
    // Main loop
    while true:
        // Check for incoming packets
        let packet = wifi_receive_nonblock()
        if packet.len() > 0:
            handle_packet(packet)
        
        // Check heartbeat timeout
        if (get_tick_ms() - state.last_heartbeat) > HEARTBEAT_TIMEOUT_MS:
            emergency_stop()
        
        // Smooth animation (interpolate if dancing)
        if state.is_dancing:
            interpolate_pose()
        
        delay_ms(20)  // 50 Hz
    
    return 0

fn init_servos():
    for i in 0..4:
        init_pwm(SERVOS[i].pin, 50)  // 50 Hz servo frequency
        set_pwm_us(SERVOS[i].pin, SERVOS[i].neutral_us)

fn init_leds():
    init_pwm(4, 1000)  // RGB LED on pin 4
    set_rgb(0, 0, 0)

fn init_wifi():
    wifi_connected = wifi_connect(WIFI_SSID, WIFI_PASS)
    if not wifi_connected:
        // Blink red LED to indicate failure
        while true:
            set_rgb(255, 0, 0)
            delay_ms(500)
            set_rgb(0, 0, 0)
            delay_ms(500)

fn init_timer():
    // Configure PIT for 1ms interrupts
    let divisor = 1193182 / 1000
    port_write8(0x43, 0x36)
    port_write8(0x40, (divisor & 0xFF) as u8)
    port_write8(0x40, ((divisor >> 8) & 0xFF) as u8)
    enable_interrupts()

// ============================================================================
// Command Handling
// ============================================================================

fn handle_packet(packet: ptr<u8>):
    let len = packet[1]
    if len < 4:
        return
    
    let cmd = packet[3]
    state.last_heartbeat = get_tick_ms()
    
    switch (cmd):
        case CommandType.Ping:
            send_pong()
        
        case CommandType.Pose:
            if len >= 8:
                let angles: array<u8, 4> = [packet[4], packet[5], packet[6], packet[7]]
                state.angles = angles
                state.is_dancing = false
                execute_pose(state.angles, state.led_rgb)
        
        case CommandType.Motion:
            if len >= 12:
                let angles1: array<u8, 4> = [packet[4], packet[5], packet[6], packet[7]]
                let angles2: array<u8, 4> = [packet[8], packet[9], packet[10], packet[11]]
                let duration = (packet[12] as u16) << 8 | packet[13]
                start_motion(angles1, angles2, duration)
        
        case CommandType.Sequence:
            let seq_name = get_string(&packet[4])
            let opt = dance_lookup(seq_name)
            if opt.is_some():
                start_sequence(opt.unwrap())
        
        case CommandType.Stop:
            emergency_stop()
        
        case CommandType.Sleep:
            enter_sleep_mode()
        
        case CommandType.Wake:
            exit_sleep_mode()
        
        default:
            // Unknown command
            send_error(0x01)

fn start_motion(angles1: array<u8, 4>, angles2: array<u8, 4>, duration_ms: u16):
    state.is_dancing = true
    // Store motion parameters for interpolation
    motion_start_angles = angles1
    motion_target_angles = angles2
    motion_duration = duration_ms
    motion_start_time = get_tick_ms()

fn interpolate_pose():
    if not state.is_dancing:
        return
    
    let elapsed = get_tick_ms() - motion_start_time
    if elapsed >= motion_duration:
        state.angles = motion_target_angles
        state.is_dancing = false
        execute_pose(state.angles, state.led_rgb)
        return
    
    let t = (elapsed as f32) / (motion_duration as f32)
    for i in 0..4:
        let start = motion_start_angles[i] as i16
        let target = motion_target_angles[i] as i16
        let current = start + ((target - start) as f32 * t) as i16
        state.angles[i] = current as u8
    
    execute_pose(state.angles, state.led_rgb)

fn execute_pose(angles: array<u8, 4>, rgb: array<u8, 3>):
    for i in 0..4:
        let pulse_us = SERVOS[i].min_us + (angles[i] as u16) * 
                       (SERVOS[i].max_us - SERVOS[i].min_us) / 255
        set_pwm_us(SERVOS[i].pin, pulse_us)
    
    set_rgb(rgb[0], rgb[1], rgb[2])

// ============================================================================
// Safety
// ============================================================================

#[interrupt(priority = 0)]
fn safety_monitor():
    // Check Wi-Fi connection
    if not wifi_is_connected():
        emergency_stop()
    
    // Check battery (if ADC available)
    let battery_mv = read_adc(ADC_BATTERY)
    if battery_mv < 3500:  // 3.5V critical
        set_rgb(255, 0, 0)  // Red LED warning
        if battery_mv < 3200:
            emergency_stop()
    
    // Check motor current
    let current_ma = read_adc(ADC_MOTOR)
    if current_ma > 500:
        disable_motors()
        set_rgb(255, 0, 0)
    
    // Re-enable interrupts for lower priority tasks
    enable_interrupts()

fn emergency_stop():
    disable_motors()
    set_rgb(255, 0, 0)  // Red
    state.is_dancing = false
    state.angles = [90, 0, 0, 0]
    execute_pose(state.angles, state.led_rgb)
    while true:
        halt()

// ============================================================================
// Power Management
// ============================================================================

fn enter_sleep_mode():
    // Save state
    let saved_angles = state.angles
    let saved_rgb = state.led_rgb
    
    // Disable Wi-Fi
    wifi_power_down()
    
    // Set CPU to low power
    set_cpu_freq(32)  // 32 MHz
    
    // Configure wake sources
    enable_wake_on_wifi()
    enable_wake_on_button()
    
    // Enter HLT
    while true:
        halt()
        if should_wake():
            break
    
    // Restore
    wifi_power_up()
    set_cpu_freq(CPU_FREQ_MAX)
    state.angles = saved_angles
    state.led_rgb = saved_rgb
    execute_pose(state.angles, state.led_rgb)

// ============================================================================
// Helper Functions
// ============================================================================

fn get_tick_ms() -> u32:
    // Read PIT counter (simplified)
    return ticks

static ticks: u32 = 0

#[interrupt]
fn timer_isr():
    ticks = ticks + 1
    port_write8(0x20, 0x20)  // EOI to PIC

fn set_rgb(r: u8, g: u8, b: u8):
    // PWM-driven RGB LED (common cathode)
    set_pwm_duty(4, r as u16)  // Red
    set_pwm_duty(5, g as u16)  // Green
    set_pwm_duty(6, b as u16)  // Blue

fn disable_motors():
    // Set motor driver enable pins low
    port_write8(MOTOR_ENABLE_PORT, 0)
```

### 8.2 Scheduler (Complete)

```lowl
// scheduler.lowl - Cooperative Round-Robin Scheduler

struct TaskControlBlock:
    name: string
    entry: fn() -> u32
    stack_top: u64
    state: u8  // 0=stopped, 1=runnable, 2=sleeping, 3=zombie
    priority: u8
    wake_time: u32
    next: ptr<TaskControlBlock>

static task_list: ptr<TaskControlBlock> = null
static current_task: ptr<TaskControlBlock> = null
static next_pid: u64 = 1

fn create_task(name: string, entry: fn() -> u32, priority: u8) -> u64:
    let tcb = physical_alloc(sizeof(TaskControlBlock), 8) as ptr<TaskControlBlock>
    tcb.name = name
    tcb.entry = entry
    tcb.state = 1  // runnable
    tcb.priority = priority
    tcb.wake_time = 0
    
    // Allocate stack (8 KB)
    let stack = physical_alloc(8192, 16) as u64
    tcb.stack_top = stack + 8192 - 8
    
    // Add to list
    tcb.next = task_list
    task_list = tcb
    
    return next_pid
    next_pid = next_pid + 1

fn scheduler_start():
    if task_list == null:
        // Idle task
        create_task("idle", idle_task, 0)
    
    current_task = task_list
    
    while true:
        if current_task.state == 1:  // runnable
            run_task(current_task)
        
        current_task = current_task.next
        if current_task == null:
            current_task = task_list
        
        pause()  // Yield

fn run_task(tcb: ptr<TaskControlBlock>):
    // Save current context (implemented in assembly)
    asm("
        push rax
        push rbx
        push rcx
        push rdx
        push rsi
        push rdi
        push r8
        push r9
        push r10
        push r11
        push r12
        push r13
        push r14
        push r15
        push rbp
    ")
    
    // Switch to task's stack
    let saved_rsp: u64
    asm("mov %0, rsp" : "=r"(saved_rsp))
    tcb.stack_top = saved_rsp  // Save current
    asm("mov rsp, %0" : : "r"(tcb.stack_top))
    
    // Call task entry point
    let result = tcb.entry()
    
    // If returns, task is done
    tcb.state = 3  // zombie
    
    // Restore previous task
    asm("mov rsp, %0" : : "r"(saved_rsp))
    asm("
        pop rbp
        pop r15
        pop r14
        pop r13
        pop r12
        pop r11
        pop r10
        pop r9
        pop r8
        pop rdi
        pop rsi
        pop rdx
        pop rcx
        pop rbx
        pop rax
        ret
    ")

fn idle_task() -> u32:
    while true:
        halt()  // Wait for interrupt
    return 0
```

```lowl
fn udp_send(ip: u32, port: u16, data: ptr<u8>, len: u64) -> bool:
    // Format AT command for UDP send
    let cmd = "AT+CIPSEND=" + ip.to_string() + "," + port.to_string() + "," + len.to_string() + "\r\n"

    delay_ms(100)
    uart_write_bytes(data, len)

    return response.contains("SEND OK")
```

## Appendix C: Glossary

**Algorithmic Movement:** The core philosophy that all robotic motion is the direct execution of an algorithm.

**AlgorithmicOS:** The operating system for robots, written in lowL.

**BlockArray:** A SIMD-optimized dynamic array container.

**Cognitive Workflow:** A high-level process managed by the AI server.

**Data Section:** A compile-time declaration of structured data (CSV, JSON, etc.).

**Embodied AI:** The combination of an AI server (mind) with a robot (body).

**lowL:** The systems programming language for Algorithmic Movement.

**MMIO (Memory-Mapped I/O):** Hardware registers accessed as memory addresses.

**Option:** A type representing an optional value (Some or None).

**PhysicalAllocator:** The physical memory manager.

**Red-Black Tree (RB Map):** A self-balancing binary search tree for associative containers.

**SIMD (Single Instruction, Multiple Data):** Parallel processing using vector registers.

**Thin Client Robot:** A robot with minimal onboard intelligence, relying on a central server.

| `undefined reference` | Link missing object file |

### D.3 Runtime Issues

| Issue | Debug Command |
|-------|---------------|
| Watchdog timeout | Send ping every 5 seconds |
| Motor overload | Check for physical obstruction |
| Battery low | Monitor ADC reading |
| Memory corruption | Enable heap canary (`-DHEAP_CHECK`) |
